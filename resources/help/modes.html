显示模式说明 Display Modes

ComAssistant 提供四种显示模式，适用于不同的使用场景。可在工具栏的"模式"下拉框中切换。

ComAssistant provides four display modes for different use cases. Switch via the "Mode" dropdown in toolbar.

1. 串口模式 Serial Mode

用途 / Purpose: 标准串口通信模式，适合日常收发数据。

Standard serial communication mode for everyday data transmission.

功能特点 Features:

  • 多标签接收区（文本/十六进制/过滤）- Multi-tab receive area (Text/Hex/Filter)
  • 支持文本和HEX格式发送 - Support text and HEX format sending
  • 支持循环发送 - Auto-send loop support
  • 支持追加换行符 - Append newline option
  • 关键词高亮功能 - Keyword highlighting
适用场景: AT指令调试、传感器数据读取、普通串口通信
Best for: AT command debugging, sensor data reading, general serial communication

2. 终端模式 Terminal Mode

用途 / Purpose: 专业VT100终端仿真器，支持完整ANSI转义序列，可与Linux/RTOS等系统进行交互式命令行操作。

Professional VT100 terminal emulator with full ANSI escape sequence support for interactive command line operation with Linux/RTOS systems.

功能特点 Features:

  • 完整VT100/ANSI支持 - 光标定位、屏幕清除、滚动区域、颜色控制
  • Full VT100/ANSI support - Cursor positioning, screen clearing, scroll regions, color control
  • 支持vim/nano等终端编辑器 - Supports vim/nano and other terminal editors
  • 命令历史记录（上/下键切换）- Command history (Up/Down keys)
  • Tab键智能补全（本地命令优先，无匹配时发送到设备）- Smart Tab completion (local commands first, sends to device if no match)
  • 本地回显选项 - Local echo option
  • 日志记录功能 - Log recording feature

支持的ANSI控制序列:

  • 光标控制: CUP(定位)、CUU/CUD/CUF/CUB(移动)、保存/恢复位置
  • 屏幕控制: ED(清屏)、EL(清行)、滚动
  • 颜色支持: 16色、256色、真彩色(RGB)
  • 文本样式: 粗体、斜体、下划线、闪烁、反显

字体设置:

在工具栏"字号"处可调整终端字体大小（6-72pt）。推荐使用等宽字体如Consolas。

Adjust terminal font size (6-72pt) in toolbar "Font Size". Monospace fonts like Consolas recommended.

Tab智能补全 Smart Tab Completion:

工作原理 / How it works:

  1. 首先在本地命令列表中查找匹配 - First searches local command list for matches
  2. 连续按Tab循环切换匹配项 - Press Tab repeatedly to cycle through matches
  3. 如果无本地匹配，发送Tab到远程设备 - If no local match, sends Tab to remote device

编辑命令列表 / Edit command list:

点击工具栏"编辑命令"按钮可自定义本地命令列表。默认包含常用命令：help, ls, cd, pwd, cat, echo, reboot, reset, version, status, clear, exit, quit

Click "Edit Commands" button in toolbar to customize local command list. Default commands include: help, ls, cd, pwd, cat, echo, reboot, reset, version, status, clear, exit, quit

注意: 控制字符是不可打印的，发送后设备回显可能看起来是空的。
Note: Control characters are non-printable, device echo may appear empty.
适用场景: Linux/RTOS命令行交互、ESP32/STM32 Shell调试、路由器/交换机配置
Best for: Linux/RTOS command line, ESP32/STM32 Shell debugging, router/switch configuration

3. 帧模式 Frame Mode

用途 / Purpose: 按帧解析数据，适合自定义协议调试。

Parse data by frames, suitable for custom protocol debugging.

功能特点 Features:

  • 自定义帧头帧尾 - Custom frame header/footer
  • 帧校验支持（XOR/SUM/CRC16）- Frame checksum (XOR/SUM/CRC16)
  • 帧列表显示有效/无效状态 - Frame list shows valid/invalid status
  • 帧详情查看（HEX/ASCII）- Frame detail view (HEX/ASCII)
  • 超时自动处理不完整帧 - Timeout handling for incomplete frames

帧配置说明 Frame Configuration:

参数 Parameter 默认值 Default 说明 Description
帧头 Header AA 帧起始标志 - Frame start marker
帧尾 Footer 55 帧结束标志 - Frame end marker
校验 Checksum 无 None 可选XOR/SUM/CRC16

帧状态说明:

  • ✓ 有效帧 - 帧头帧尾正确，校验通过
  • ✗ 无效帧 - 帧头帧尾错误、超时或校验失败
适用场景: Modbus调试、自定义二进制协议、传感器数据帧解析
Best for: Modbus debugging, custom binary protocols, sensor data frame parsing

4. 调试模式 Debug Mode

用途 / Purpose: 高级调试功能，支持断点、时间差分析。

Advanced debugging with breakpoints and time difference analysis.

功能特点 Features:

  • 数据记录表格（按行显示）- Data record table (line by line)
  • TX/RX方向标记 - TX/RX direction markers
  • 时间差计算（相邻数据间隔）- Time difference calculation
  • 断点功能（匹配特定数据时暂停）- Breakpoint (pause on matching data)
  • 暂停/单步功能 - Pause/Step functions
  • 监视变量显示 - Watch variables display
  • 数据过滤功能 - Data filtering

工具栏按钮说明:

按钮 Button 功能 Function
暂停/继续 暂停数据更新，可查看历史数据 - Pause updates to review history
步进 暂停状态下逐条显示数据 - Show data one by one when paused
添加断点 设置数据断点，匹配时自动暂停 - Set data breakpoint
HEX 切换HEX/文本显示格式 - Toggle HEX/Text display
时间差 显示/隐藏时间差列 - Show/hide time difference column

断点使用方法:

  1. 点击"添加断点"按钮
  2. 输入要匹配的模式（支持HEX或文本）
  3. 当接收数据包含该模式时，自动暂停

示例：输入 ERROR 作为断点，收到包含 "ERROR" 的数据时暂停。

适用场景: 协议分析、时序调试、异常数据捕获
Best for: Protocol analysis, timing debugging, exception capture
