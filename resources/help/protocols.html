协议说明 Protocol Documentation

中文：串口调试助手支持多种协议，包括数据显示协议和绘图协议。

English: ComAssistant supports multiple protocols, including data display protocols and plotting protocols.

1. 数据显示协议 / Data Display Protocols

1.1 原始数据 (Raw)

说明 / Description: 直接显示接收到的原始字节数据 / Display raw byte data as received

用途 / Usage: 适用于所有场景 / Suitable for all scenarios

1.2 ASCII协议 (ASCII)

说明 / Description: 将数据按ASCII字符显示 / Display data as ASCII characters

用途 / Usage: 文本数据传输 / Text data transmission

1.3 HEX协议 (Hexadecimal)

说明 / Description: 将数据按十六进制格式显示 / Display data in hexadecimal format

格式 / Format: AA BB CC DD

用途 / Usage: 二进制数据调试 / Binary data debugging

1.4 Modbus协议 (Modbus RTU)

说明 / Description: Modbus RTU工业通信协议 / Modbus RTU industrial protocol

格式 / Format: [地址][功能码][数据][CRC16]

用途 / Usage: 工业设备通信 / Industrial device communication

示例 / Example:
读保持寄存器 / Read Holding Registers:
01 03 00 00 00 0A C5 CD
  • 01 - 从站地址 / Slave Address
  • 03 - 功能码（读保持寄存器）/ Function Code (Read Holding Registers)
  • 00 00 - 起始地址 / Start Address
  • 00 0A - 寄存器数量 / Register Count
  • C5 CD - CRC16校验 / CRC16 Checksum

1.5 EasyHEX协议 (Easy Hexadecimal)

说明 / Description: 简易十六进制协议，支持灵活配置帧结构 / Easy hexadecimal protocol with flexible frame structure

帧格式 / Frame Format:

结构 / Structure 说明 / Description
帧头 / Frame Header 默认: AA 55，可配置 / Default: AA 55, configurable
长度字段 / Length Field 1-2字节，指示后续数据长度 / 1-2 bytes, indicates payload length
数据 / Data 有效载荷数据 / Payload data
校验和 / Checksum 支持 SUM8/XOR8/CRC8 / Supports SUM8/XOR8/CRC8
帧尾 / Frame Tail 可选，可配置 / Optional, configurable
示例 / Example:
发送数据 "Hello" / Send data "Hello":
AA 55 06 48 65 6C 6C 6F E5
  • AA 55 - 帧头 / Frame Header
  • 06 - 长度（5字节数据 + 1字节校验）/ Length (5 bytes data + 1 byte checksum)
  • 48 65 6C 6C 6F - "Hello" 的ASCII码 / ASCII of "Hello"
  • E5 - SUM8校验和 / SUM8 Checksum

校验算法 / Checksum Algorithms:

类型 / Type 算法 / Algorithm
SUM8 累加和取低8位 / Sum of bytes, lower 8 bits
XOR8 所有字节异或 / XOR of all bytes
CRC8 CRC-8 (多项式: 0x07) / CRC-8 (polynomial: 0x07)

适用场景 / Use Cases:

  • 自定义嵌入式设备通信 / Custom embedded device communication
  • 简单可靠的数据传输 / Simple and reliable data transmission
  • 需要帧校验的场景 / Scenarios requiring frame verification

2. 绘图协议 / Plotting Protocols

中文：绘图协议用于实时绘制数据曲线，适合传感器数据可视化。

English: Plotting protocols are used for real-time data plotting, suitable for sensor data visualization.

2.1 TEXT 绘图协议

格式 / Format:

{窗口ID}数据1,数据2,数据3,...
{windowId}data1,data2,data3,...

说明 / Description:

字段 / Field 说明 / Description
{窗口ID} 指定绘图窗口ID，如 {plot1} / Specify plot window ID
数据 / data 逗号分隔的数值，每个数值对应一条曲线 / Comma-separated values, each value represents a curve
示例 / Example: 
{plot1}1.23,4.56,7.89
{plot1}2.34,5.67,8.90
{plot1}3.45,6.78,9.01

效果 / Result:

  • 创建名为 plot1 的绘图窗口 / Create a plot window named plot1
  • 绘制3条曲线 / Plot 3 curves
  • X轴自动递增（0, 1, 2, ...）/ X-axis auto increments (0, 1, 2, ...)
  • Y轴分别为 (1.23, 4.56, 7.89), (2.34, 5.67, 8.90), ... / Y-axis values are (1.23, 4.56, 7.89), ...

适用场景 / Use Cases:

  • 温度传感器数据（多通道）/ Temperature sensor data (multi-channel)
  • 电压/电流监测 / Voltage/Current monitoring
  • 加速度计数据（X, Y, Z轴）/ Accelerometer data (X, Y, Z axes)

2.2 STAMP 绘图协议（带时间戳）

格式 / Format:

<时间戳>{窗口ID}数据1,数据2,...
<timestamp>{windowId}data1,data2,...

说明 / Description:

字段 / Field 说明 / Description
<时间戳> Unix时间戳（毫秒），作为X轴 / Unix timestamp (ms), used as X-axis
{窗口ID} 绘图窗口ID / Plot window ID
数据 / data Y轴数值 / Y-axis values
示例 / Example: 
<1000>{plot2}10.5,20.3
<2000>{plot2}11.2,21.8
<3000>{plot2}12.7,22.1

效果 / Result:

  • X轴为时间戳（1000, 2000, 3000）/ X-axis is timestamp (1000, 2000, 3000)
  • Y轴为数据值 / Y-axis is data values
  • 支持非均匀采样 / Supports non-uniform sampling

适用场景 / Use Cases:

  • GPS轨迹记录 / GPS trajectory recording
  • 事件驱动的数据采集 / Event-driven data acquisition
  • 非周期性采样 / Non-periodic sampling

2.3 CSV 绘图协议

格式 / Format:

窗口ID,X值,Y1,Y2,Y3,...
windowId,xValue,y1,y2,y3,...

说明 / Description:

列 / Column 说明 / Description
第1列 / Column 1 窗口ID / Window ID
第2列 / Column 2 X轴值 / X-axis value
后续列 / Following columns Y轴值（多条曲线）/ Y-axis values (multiple curves)
示例 / Example: 
plot3,0.1,100,200,300
plot3,0.2,110,210,310
plot3,0.3,120,220,320

效果 / Result:

  • X轴为 0.1, 0.2, 0.3 / X-axis is 0.1, 0.2, 0.3
  • 曲线1: (0.1,100), (0.2,110), (0.3,120)
  • 曲线2: (0.1,200), (0.2,210), (0.3,220)
  • 曲线3: (0.1,300), (0.2,310), (0.3,320)

适用场景 / Use Cases:

  • 频率响应曲线 / Frequency response curve
  • 校准曲线 / Calibration curve
  • 自定义X轴刻度 / Custom X-axis scale

3. 协议选择指南 / Protocol Selection Guide

场景 / Scenario 推荐协议 / Recommended Protocol 原因 / Reason
文本通信 / Text communication ASCII 直观易读 / Intuitive and readable
二进制调试 / Binary debugging HEX 精确显示字节值 / Precise byte values
工业设备 / Industrial devices Modbus 标准工业协议 / Standard industrial protocol
实时曲线（简单）/ Real-time curve (simple) TEXT 格式简单 / Simple format
时序数据 / Time-series data STAMP 包含时间信息 / Contains time information
完整XY数据 / Complete XY data CSV 灵活性高 / High flexibility

4. 注意事项 / Notes

  • 编码格式 / Encoding: 所有文本协议使用UTF-8编码 / All text protocols use UTF-8 encoding
  • 数据分隔 / Data Separator: CSV使用逗号分隔，不支持空格 / CSV uses comma separator, spaces not supported
  • 窗口ID / Window ID: 窗口ID必须唯一，建议使用字母数字组合 / Window ID must be unique, alphanumeric recommended
  • 性能优化 / Performance: 高速数据（>100Hz）建议使用数据抽稀 / High-speed data (>100Hz) should use data decimation
  • 曲线数量 / Curve Count: 单窗口建议不超过16条曲线 / Single window should have ≤16 curves
