# 常见问题 (FAQ)

## 连接问题

### Q: 串口打不开，提示"设备正在使用中"？

**A**: 可能的原因和解决方案：
1. 其他程序正在使用该串口 → 关闭其他串口工具
2. 串口设备未正确连接 → 检查 USB 连接
3. 驱动程序问题 → 重新安装驱动

### Q: 找不到串口设备？

**A**:
1. 确认设备已正确连接
2. 检查设备管理器中是否有 COM 端口
3. 点击"刷新"按钮重新扫描
4. 尝试重新插拔设备

### Q: 连接后无法收发数据？

**A**:
1. 确认波特率设置正确
2. 检查数据位、停止位、校验位设置
3. 确认设备端程序正常运行
4. 尝试交换 TX/RX 线（如果是手工连接）

## 数据问题

### Q: 接收到的数据是乱码？

**A**:
1. 波特率不匹配 → 调整为正确的波特率
2. 编码问题 → 尝试切换 ASCII/HEX 显示
3. 数据位设置错误 → 检查数据位设置（通常为 8 位）

### Q: 发送的数据设备收不到？

**A**:
1. 检查流控制设置
2. 确认硬件连接正确
3. 检查设备端接收程序

### Q: 数据显示不完整？

**A**:
1. 缓冲区溢出 → 降低发送频率
2. 显示刷新不及时 → 关闭时间戳显示
3. 数据量过大 → 清空接收区后重试

## 绘图问题

### Q: 绘图窗口不显示曲线？

**A**:
1. 确认选择了正确的绘图协议
2. 检查数据格式是否符合协议要求
3. 查看接收区是否有数据

### Q: 绘图卡顿？

**A**:
1. 减少数据点数量
2. 降低发送频率
3. 关闭其他绘图窗口
4. 启用 OpenGL 加速

### Q: 曲线数据异常？

**A**:
1. 检查数据格式是否正确
2. 确认数值范围合理
3. 查看是否有数据解析错误

## 性能问题

### Q: 程序启动慢？

**A**:
1. 清理日志文件
2. 减少快捷发送条目
3. 关闭不需要的功能

### Q: 内存占用高？

**A**:
1. 定期清空接收区
2. 减少绘图窗口数量
3. 限制数据缓存大小

### Q: CPU 占用高？

**A**:
1. 降低数据刷新频率
2. 关闭实时高亮
3. 减少同时打开的窗口

## 界面问题

### Q: 界面显示模糊？

**A**:
1. 检查 Windows 缩放设置
2. 右键程序 → 属性 → 兼容性 → 更改高 DPI 设置

### Q: 语言切换后部分文字未更新？

**A**:
1. 重启程序
2. 检查翻译文件是否完整

### Q: 主题切换不生效？

**A**:
1. 确认主题文件存在
2. 重启程序

## 脚本问题

### Q: Lua 脚本无法运行？

**A**:
1. 检查语法错误
2. 查看输出窗口的错误信息
3. 确认 API 调用正确

### Q: 脚本执行超时？

**A**:
1. 检查是否有死循环
2. 优化脚本逻辑
3. 增加超时时间

## 其他问题

### Q: 如何导出所有配置？

**A**:
1. 备份 `%APPDATA%\ComAssistant\config.ini`
2. 备份 `%APPDATA%\ComAssistant\quicksend.json`

### Q: 如何重置所有设置？

**A**:
1. 关闭程序
2. 删除 `%APPDATA%\ComAssistant\config.ini`
3. 重新启动程序

### Q: 如何报告 Bug？

**A**:
1. 收集日志文件：`%APPDATA%\ComAssistant\logs\`
2. 描述复现步骤
3. 提交到 GitHub Issues
